# AI-Ready Design System — Aadith V A Portfolio

This folder makes the portfolio design system understandable to both humans and AI
assistants. It combines **atomic design** with a 3-layer, machine-readable retrieval
pipeline inspired by the AI-ready design-system methodology — all generated from the
**Astro** source so it never drifts.

## Architecture overview

### Atomic design layers

- **Tokens**: framework-agnostic `--ds-*` custom properties for color, typography,
  spacing, radii, container sizing, motion, and breakpoints. Sources:
  `design-system/tokens.css` / `tokens.json`, consumed by `src/styles/global.css`.
- **Atoms** (`src/components/atoms`): `Reveal` (scroll entrance motion), `Line`
  (animated divider), `SectionLabel` (uppercase eyebrow).
- **Molecules** (`src/components/molecules`): `CaseStudySection` (data-driven case-study
  blocks), `HoverRevealList` (numbered work index), `LabCard` (Lab grid card).
- **Organisms** (`src/components/organisms`): `Header`, `Footer`, `Hero`, `FeaturedWork`,
  `ProjectHero`, `ProjectNav`.
- **Layout** (`src/layouts`): `BaseLayout` — the shell with head, fonts, header/footer,
  and the global reveal / mobile-menu / route-line JS.
- **Pages / routes** (`src/pages`): home, about, work, work/[slug], lab, lab/[slug],
  adventures, writing, contact.

Entrance motion is driven by a single IntersectionObserver in `BaseLayout`
(`.reveal` / `.line` / `.routemap` → `.is-visible`), replacing the original
GSAP / Framer Motion / Lenis stack and honouring `prefers-reduced-motion`.

### The 3 AI layers

1. **Index — What/Where**
   - File: `design-system/ai/index.json`
   - Generated by: `python3 design-system/ai/codebase-index.py`
   - Maps every component/route to its relative path, tier, imports, and local
     component/data dependencies.
   - Keys are **full relative paths**, not filename stems, so collisions like the two
     `[slug].astro` files (`lab/[slug]` and `work/[slug]`) never overwrite each other.

2. **Metadata — How/Why**
   - Files: `design-system/ai/metadata/*.json` (one per component)
   - Generated by: `python3 design-system/ai/component-metadata.py`
   - Structural fields (tier, props, dependencies, used-by) are derived from source +
     `index.json`; curated What / Why / when-to-use prose lives in the generator's
     `CONTENT` map. This captures design intent and usage judgment, not just types.

3. **Reasoning / Composer — Selection logic**
   - File: `design-system/ai/ds-composer.md`
   - Guides AI assistants to choose layout/organisms first, then molecules, then atoms,
     and only build custom UI when existing patterns do not fit.

## How to use this system

1. Start with `design-system/CLAUDE.md` for AI operating instructions.
2. Read `design-system/ai/index.json` to understand what exists and where.
3. Open relevant `design-system/ai/metadata/*.json` for usage rules and anti-patterns.
4. Apply `design-system/ai/ds-composer.md` before creating or changing UI.
5. Use `design-system/tokens.css` / `tokens.json` as the source of design values.

## Maintenance plan

### Regeneration pipeline

When components, routes, or data change, re-run the three generators in order:

```bash
python3 design-system/ai/codebase-index.py      # WHAT/WHERE  -> ai/index.json
python3 design-system/ai/component-metadata.py   # HOW/WHY     -> ai/metadata/*.json
python3 design-system/ai/adoption-report.py      # analysis    -> component-inventory.md + adoption-report.md
```

`adoption-report.py` derives the human-readable `component-inventory.md` and
`adoption-report.md` (utilization, tier distribution, metadata coverage, risk flags)
entirely from the index + metadata, so those docs stay accurate automatically.

### Recommended CI step

1. Run all three generators.
2. Fail the build if any generated file has unstaged changes (drift guard).
3. Publish the refreshed adoption report on design-system changes.

### Version stamping

Every generated artifact carries `generatedAt` / `lastUpdated`. Treat stale
index/metadata as advisory and re-run the generators when code has changed.

## Current caveats

- This foundation mirrors `src/`; it documents and indexes but does not alter runtime
  behavior beyond the generated `.md`/`.json` artifacts in this folder.
- Utilization in the adoption report is static-import based; dynamic or runtime-only
  references may need manual review.
